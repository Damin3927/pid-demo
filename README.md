# PID制御 インタラクティブ入門

1関節ロボットアームを題材に、PID制御を **P → PD → PID** と一段ずつ積み上げて学ぶ
インタラクティブ Web デモです。各項（P・I・D）が「いま何をしているか」をリアルタイムの
トルク内訳バーで可視化し、各段階で起きる現象を体験できます。

## 学べること

| ステップ | 体験する現象 |
| --- | --- |
| **P制御** | 比例ゲインだけ。速いほど振動し、重力のせいで目標手前で止まる（**定常偏差**） |
| **PD制御** | 微分でブレーキ。振動が収まり大きな Kp が使える。だが定常偏差は残る |
| **PID制御** | 積分が一定トルクを作り出し、**定常偏差が消える**。ワインドアップも体験可能 |
| **自由実験** | すべてのつまみを自由に。速さ・安定性・正確さのトレードオフを探る |

## 物理モデル

ピボットまわりの剛体回転として、角度依存の重力負荷を外乱に持つ系を扱います。

```
J·θ'' = τ_motor − b·θ' − m·g·L·cos(θ)
τ_motor = clamp(Kp·e + Ki·∫e dt + Kd·de/dt,  ±τ_max)
```

`m·g·L·cos(θ)`（重力負荷）は目標角度でもゼロにならないため、P / PD では誤差を残さないと
釣り合わず、これが定常偏差として現れます。積分項がこの保持トルクを肩代わりして偏差を消します。

- 数値積分: セミインプリシット（シンプレクティック）オイラー、固定ステップ `dt = 2ms`
- 微分項は測定値（角速度）に対して作用させ、目標変更時の微分キックを回避
- 積分にはアンチワインドアップ（条件付き積分）を実装。PID/自由実験で ON/OFF 切替可能

## 開発

```bash
pnpm install
pnpm dev        # 開発サーバ
pnpm build      # 型チェック + 本番ビルド
pnpm preview    # ビルド結果のプレビュー
pnpm typecheck  # 型チェックのみ
```

## 技術スタック

Vite + React + TypeScript + Tailwind CSS v4。可視化（アーム・応答グラフ）は依存を増やさず
Canvas に直接描画しています。シミュレーションのコアは `src/sim/pid.ts`（純粋関数）に分離。

## 構成

```
src/
  sim/
    pid.ts          物理 + PID の純粋なステップ計算
    usePidSim.ts    requestAnimationFrame で回す React フック（指標計算込み）
    config.ts       プラント定数・ゲイン範囲
  components/        ArmCanvas / ResponseChart / GainSlider / TermBar / ...
  steps/steps.tsx    各ステップの解説文と既定ゲイン
  App.tsx            ステッパーと画面遷移
```
